## 🚀 What's New in v1.1

### 1. **ClipboardWatcher** - Auto-Detect AI Responses

**Location:** `SovereignIDE.Core/Services/ClipboardWatcher.cs`

**Features:**
- Monitors clipboard every 500ms
- Detects AI response patterns automatically
- Fires event when response detected
- Respects pause button (doesn't auto-execute)
- Can be enabled/disabled in settings

**How it Works:**
```
1. You copy DeepSeek/Claude response
2. Clipboard watcher detects it (checks for code blocks, AI phrases)
3. Shows notification: "AI response detected"
4. Asks: "Process now?"
5. You click Yes → files extracted, commands queued
```

**Detection Heuristics:**
- Contains code blocks (```)
- Contains AI phrases ("Here's the", "I'll create", etc.)
- Contains JSON manifest structure
- Has code-like content (classes, functions, namespaces)

**Why This Matters:**
No more manual `Ctrl+V`. Just copy in browser, switch to Sovereign IDE, click Yes.

### 2. **ManifestManager** - Smart Persistence

**Location:** `SovereignIDE.Core/Services/ManifestManager.cs`

**Features:**
- Auto-save every 30 seconds (configurable)
- Backup before overwrite (keeps 10 most recent)
- Atomic writes (temp file → move)
- Recovery from corruption (loads latest backup)
- Version validation
- Dirty flag tracking

**Auto-Save Workflow:**
```
1. You make changes (add file, approve command)
2. Manifest marked as "dirty"
3. After 30 seconds → auto-saves
4. Backup created: .manifest-backups/manifest-20260328-143000.json
5. Old manifest overwritten atomically
6. Status bar: "📝 Manifest auto-saved"
```

**Recovery on Corruption:**
```
1. manifest.json corrupted (power loss, disk error)
2. Load() detects JSON parse error
3. Scans .manifest-backups/
4. Loads most recent valid backup
5. Shows: "⚠️ Manifest was corrupted, recovered from backup"
```

**Why This Matters:**
Never lose work. Even if you forget to save manually, auto-save has your back.

## 🎯 Updated Features in MainForm

### Integration Points

**On Startup:**
1. Load configuration from `%APPDATA%\SovereignIDE\config.json`
2. Restore window layout (size, position, panel widths)
3. Initialize keyboard shortcut manager
4. Start clipboard watcher (if enabled)

**On Project Open:**
1. Initialize ManifestManager with auto-save
2. Try to load existing `manifest.json`
3. If not found, create new manifest
4. Add project to recent projects list
5. Save configuration

**On Project Work:**
1. Clipboard detects response → asks to process
2. Files/commands extracted → manifest marked dirty
3. After 30 seconds → auto-save with backup
4. Error occurs → ErrorDialog with auto-copy

**On Close:**
1. Save window layout to config
2. Dispose clipboard watcher
3. ManifestManager final save (if dirty)
4. Clean shutdown

## 🔧 How to Use New Features

### Enable/Disable Clipboard Watcher

**Option 1:** Edit config manually
```json
// %APPDATA%\SovereignIDE\config.json
{
  "preferences": {
    "clipboardWatcherEnabled": false  // ← Set to false
  }
}
```

**Option 2:** (Future v1.2) Settings dialog in UI

### Adjust Auto-Save Interval

```json
{
  "preferences": {
    "autoSaveEnabled": true,
    "autoSaveIntervalSeconds": 60  // ← Change from 30 to 60
  }
}
```

### Restore from Backup

If manifest corrupted:
1. Look in `.manifest-backups/` folder
2. Find most recent: `manifest-20260328-143000.json`
3. Copy to `manifest.json`
4. Reopen project

Or just load again — recovery is automatic.

### View All Shortcuts

Press `F1` anywhere in the app → Shortcut Help Dialog appears.

## 🐛 Known Limitations

### 1. API Keys Not Encrypted
**Status:** Stored as plaintext in config.json  
**Risk:** Low (local user account only)  
**Fix:** TODO v1.2 - Use Windows DPAPI encryption  
**Workaround:** Don't store API keys if on shared computer

### 2. Clipboard Watcher False Positives
**Issue:** Might detect non-AI text as response  
**Impact:** Low (just shows notification, doesn't auto-execute)  
**Fix:** Improve heuristics in future  
**Workaround:** Click "No" if false detection

### 3. No Settings UI
**Issue:** Must edit config.json manually  
**Impact:** Medium (not user-friendly for non-devs)  
**Fix:** Planned v1.2 - Settings dialog  
**Workaround:** Edit JSON (it's documented)

## 📈 Performance Impact

**Clipboard Watcher:**
- Polls every 500ms
- CPU usage: <0.1%
- Memory: ~1MB

**Auto-Save:**
- Runs every 30 seconds (only if dirty)
- CPU spike: ~50ms
- Disk I/O: ~10KB-1MB depending on manifest size

**Backup System:**
- Creates backup on every save
- Keeps 10 most recent
- Cleanup: automatic
- Disk usage: ~10 backups × manifest size (usually <10MB total)

**Overall:** Negligible impact. You won't notice it.
