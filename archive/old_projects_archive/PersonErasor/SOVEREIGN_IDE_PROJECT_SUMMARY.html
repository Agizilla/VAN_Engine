📋 Project Overview

Mission Statement

Build a sovereign AI development environment that enables true AI-human co-evolution through persistent memory, feedback loops, and offline operation—with no vendor lock-in.

Core Philosophy

  • Persistent Memory: Context never resets. Manifest serves as shared memory across sessions.
  • Human Control: Pause button always works. All operations require approval.
  • Model Agnostic: Works with Claude, DeepSeek, Gemini, local models—your choice.
  • Offline First: Distilled models run locally. No internet required for execution.
  • Self-Improving: Learns from feedback. Every rating adjusts future behavior.
Key Insight: "Less is better." Instead of training 600B parameter models on petabytes of data, we distill reasoning primitives (how to think) into 32,000 tokens. Result: Domain-focused intelligence without bloat.

Core Components

SovereignIDE.Core (Class Library)

Models (ManifestV4.cs - 600+ lines)

  • ManifestV4, ContextInfo, FileEntry, CommandEntry, Decision, ConversationTurn
  • All schema types as C# records with immutability

Exceptions (SovereignExceptions.cs)

  • Hierarchy: SovereignException → FileOperationException, ValidationException, etc.
  • ToCopyPasteFormat() method for instant error sharing

Validation (Validators.cs)

  • PathValidator: Blocks .., absolute paths, URL encoding
  • CommandValidator: 18+ dangerous patterns (rm -rf, format, shutdown)
  • StateValidator, ContentValidator

Parsers (ResponseParser.cs)

  • Extracts code blocks, infers filenames, detects manifests
  • Handles chatty AI responses (mixed narrative + code)

FileSystem (FileManager.cs)

  • Safe I/O with validation on every operation
  • Batch operations with rollback
  • Atomic writes where possible

Execution (CommandExecution.cs)

  • CommandQueue: Approval workflow
  • CommandExecutor: Timeout, output capture
  • AutoExecutor: Pause button logic

AI (AnthropicClient.cs)

  • Full Messages API with streaming
  • Retry logic with exponential backoff
  • Rate limit handling

Services

  • ClipboardWatcher: Auto-detect AI responses (500ms polling)
  • ManifestManager: Auto-save every 30s, backup before overwrite (keeps 10)
  • ConfigurationManager: Persistent settings in %APPDATA%

Battle Rap Generator (Python)

SemanticToneDetector (Singleton)

  • Model: all-MiniLM-L6-v2 (80MB)
  • Tone prototypes: aggressive, playful, sad, hopeful, confident, sarcastic
  • Performance: 55ms per verse on CPU

BattleRapGenerator

  • N-gram model per persona with Zipf-weighted sampling
  • Semantic tone detection replaces keyword matching
  • Feedback recording: stores rating + ngrams_used
  • Weight adjustment: 5-star → 1.2x, 1-star → 0.8x, cumulative
Code Generator (Python)

FeedbackGenerator (Base Class)

  • Abstract: _tokenize(), _build_base_model()
  • Shared: feedback recording, weight adjustment, cache management

CSharpCodeGenerator

  • Tokenization: keywords preserved, identifiers abstracted as ID:X
  • Intent-based bias: frustrated → +30% keywords, confused → +20% identifiers
  • Generates fixes from error messages with context code seeding
  • Performance: 130ms cold start, 25ms warm cache

CodeValidator

  • Compilation check via dotnet build
  • Static analysis: naming conventions, missing usings
  • Validation score: 0-100 (100 = perfect, 0 = won't compile)
  • Retry logic: up to 3 attempts with error feedback

Data Flow: Code Generation Example

1. USER REPORTS ERROR
   "CS0246: Type 'ManifestV4' not found"
   ↓

2. RIGHT-BRAIN: SEMANTIC TONE DETECTION
   Input: Error message
   SemanticToneDetector → "frustrated" (confidence: 0.54)
   ↓

3. RIGHT-BRAIN: CODE GENERATION
   Input: Error + tone + codebase n-gram model
   CSharpCodeGenerator → "using SovereignIDE.Core.Models;"
   Tracks: ngrams_used for feedback
   ↓

4. LEFT-BRAIN: VALIDATION
   Input: Generated code
   CodeValidator → dotnet build
   Output: { valid: true, score: 90, warnings: 2 }
   ↓

5. RETRY LOGIC (if validation fails)
   If score < threshold:
     Error feedback → "Previous attempt failed: {error}"
     Retry generation (max 3 times)
   ↓

6. PRESENT TO USER
   Show: Generated code + validation report
   ↓

7. USER RATES (1-5)
   Rating: 5 stars
   ↓

8. FEEDBACK LOOP
   Record: { error, code, rating: 5, ngrams_used }
   Adjust: All ngrams in code → weight × 1.2
   Persist: feedback.json
   ↓

9. NEXT GENERATION
   Load model with adjusted weights
   High-rated patterns appear more often
   System has learned YOUR preferences
