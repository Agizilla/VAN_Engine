🧠 Core Concept

Upload code files → each becomes a living Citizen on a map. Houses represent projects/namespaces. Civilisation health mirrors your code's compliance, complexity and build health across 4 tiers of escalating depth.

Every file analysed against Frya's Tex (13 Directives). Clean code = green houses & prosperous citizens. Violations (telemetry, proprietary locks) trigger decay, red houses, collapse timer – fix before timer expires.

🕹️ Autonomous citizens, you refactor

No micro‑management. Your power: refactor, merge, fix code – the game reacts. Day/Night cycle separates runtime simulation from safe IDE maintenance. Git operations advance years, branches fork timelines, mods extend behaviour.

🔍 Automated Code Review Pipeline (VAN_Engine)

When you drop a source file, the TexStaticAnalyzer runs a multi‑stage review that checks each line against all integrated laws. Below is the exact sequence of validations.

1. Telemetry / Cloud Detection

Scans for telemetry, analytics, api. patterns → triggers Directive 6 (Expel Bastards).

2. Vendor Lock‑in / License Checks

Matches .lic, license, proprietary → triggers Directive 7 (No Debt Slavery).

3. Code Smells

Console.WriteLine → warning (orange house).

4. Namespace Extraction

Identifies namespace declaration → assigns citizen to a Family Tribe.

5. Line Count → Role Mapping

Lines <50 → Idle, 50‑250 → Farmer, 250‑750 → Militia, >750 → Grevetman.

6. Compliance Score

Score = 100 - (violations × 2.5). If ≥80 → green house / compliant citizen, else red house / decay timer.

7. Tribunal Acceptance

If you accept (A), the file is added to the world: resources rewarded or penalties applied. If rejected (B), no effect.

8. House Build State Update

Aggregate errors/warnings from all owned files → house colour changes; red house starts collapse timer.

🔁 Continuous Integration Loop

Every night (or when you manually fix files), the analyzer re‑scans all owned files and updates the house state. Green houses produce treasures; red houses decay until fixed.

Result: Your real codebase's health is directly reflected in the simulation – gamified technical debt and architecture validation.

  • FlushPending callback for synchronous read on demand

🏁 Endgame v2.0.0 — New Systems

Four new major systems that transform the Oera Linda Simulator into a complete moddable dev ecosystem with git-lifeline mechanics, shareable reports, scripting hooks, and community scoreboards.

🔀 Git Integration Engine

Simulation/GitIntegrationEngine.cs

Git as a core game mechanic — every git operation has a direct simulation consequence:

  • G — Commit: advances the game year by 1, adds sovereignty, logs to timeline
  • Ctrl+B — Branch: forks a parallel workspace with independent sovereign state
  • Ctrl+M — Merge: spawns 1–3 merge conflicts between citizens. Clean merge = +3 sovereignty
  • Ctrl+R — Rebase: time-travel rewind, -2 sovereignty per year displaced
  • Conflicts: two citizens fight over a file. Unresolved after 30s → both citizens decay (60s timer), -5 sovereignty
  • Click conflict label in the overlay to resolve manually
  • 15% chance of spontaneous conflict per non-main commit
  • Conflicts displayed as color-coded overlay lines (gold → red as timer expires)

📊 Project Health Report Export

Core/ReportGenerator.cs

Generate comprehensive project health reports in two formats — shareable with your team without running the simulation:

  • HTML report (H): dark-themed styled page with sovereignty, resource ledger, citizen roster, houses/build states, directive violations table, year-over-year activity chart
  • Markdown report (R): portable plain-text format, same data
  • Sovereignty score color-coded: green (≥70%), gold (≥40%), red (<40%)
  • Saved to reports/ directory with timestamp filenames
  • UI buttons: REPORT / MD REP in bottom toolbar

🧩 Scripting API / Mod Hooks

Infrastructure/ModLoader.cs

Full modding surface with zero new NuGet dependencies — uses AssemblyLoadContext for clean isolation:

  • IMod interface with 6 lifecycle hooks:
    • OnLoad(SovereignState) — initialize mod state
    • OnYearTick(int year) — react to year changes
    • OnFileAnalyzed(filePath, result) — intercept analysis
    • OnModifyResources(ref ResourcePack) — adjust resource flow
    • OnSovereigntyChange(ref double, string) — modify sovereignty deltas
    • OnDispose() — cleanup on shutdown
  • Loads all .dll files from /mods directory at startup
  • Collectible assemblies for future hot-reload
  • Error isolation: one mod's exception never crashes others or the engine
  • Status indicator: MODS: 2 ok, 0 fail

🏆 Optional Community Scoreboard

Infrastructure/ScoreboardService.cs

Strictly opt-in, no telemetry by default. Self-certifying signed score tokens:

  • ScoreToken contains: player name, year, sovereignty, language purity, total compliant lines, citizen count, house count, total wealth, total food
  • SHA256 signature: self-certifying — no server required for verification
  • Export (Ctrl+F or SCORE button): saves JSON token to scoreboard/ directory
  • Submit: SubmitToLeaderboard() POSTs to configurable URL (disabled by default)
  • Verify: VerifyToken(json) validates signature without network
  • Zero server-side tracking — the score is self-certifying
