🎮 Context & System Vision

An offline-first, professional-grade development companion and gamified CI tracker built on the .NET 8 / Raylib-CsLo powered VAN_Engine. It translates local code architecture, file dependencies, and software engineering discipline directly into a macro-simulation ecosystem with 4 tiers of escalating depth.

🧬 Tier 1 – Foundation

Save/load, file watcher, multi-workspace, camera system, blueprint overlay, particles, similarity links, tribunal, collapse/decay timers.

🏛️ Tier 2 – Features

Inspector panel, history timeline, multi-language analysis, dependency graph, duplication detector, commons zone, refactoring, directive config, phase transitions.

⚔️ Tier 3 – Advanced

Threats/CVE simulation, tech tree upgrades, GeckoShift treasure, citizen XP/skills, trade routes, event webhook.

🏁 Endgame

Git simulation engine, health report export, mod loader API, community scoreboard, contextual help overlay.

🌞 The Day/Night Dual-Phase Loop

The simulation clock runs on an explicit, user-toggled macroscopic state machine (TimePhaseController.cs) giving developers absolute control over development rhythm.

☀️ Daytime – Execution & Ingestion Mode

  • Simulation runs at normal update frame rate (advancing game years, ticking down project stability)
  • Characters actively move across the map grid to harvest snippets and resources
  • Active threat vectors (background vulnerability sweeps) creep toward your borders
  • Trade routes deliver resources, git commits advance years

🌙 Nighttime – The IDE Maintenance Grid

  • Global execution clock freezes completely (0 FPS engine drift)
  • Energy drain and system decay counters stop
  • UI triggers a midnight-blue blueprint shader layout
  • Safely drag-and-drop elements, rewrite and refactor files, update namespaces, run compile passes with zero resource loss and zero attack risk
  • Citizens animate to their home house positions

🏛️ ProjectHouse Build States (BuildState : byte)

Each ProjectHouse aggregates static diagnostic outputs from its assigned files into a real-time structural health and visual colour profile.

🟢 Success – Green House

Perfect compile state. High integrity. Continually ticks out resources. Can spawn Backyard Treasure at Citadel tier.

🟠 Warning – Orange House

Compiles successfully but contains code smells, excessive lengths, or minor anti-patterns.

🔴 Error – Red House

Broken compilation or critical compliance failure. Roof physically collapses. Triggers a 120.0-second daytime rectification window before localised system crash. Threat magnet (60% targeting bias).

🔵 Missing – Blue House

Skeletal ghost structure where local configurations reference a file that has been deleted or moved off-disk.

⚫ Empty – Black House

Newly constructed house layout with no files or namespaces assigned yet.

📋 Technical Clarity & Decisions

⏰ No Linear Clocks at Night

Time does not tick dynamically at night. Development remains a stress-free environment where refactoring happens at the player's pace.

📦 Flat Data Layouts

Explicit value structures (FileNodeAsset) and lightweight byte mappings avoid continuous memory performance degradation within the game loop.

✋ Manual Control Principle

All structural state checks, workspace shifts, and compilation loops respond entirely to local, user-driven inputs — adhering to the zero-telemetry, manual-upgrade ethos.

🔒 lock-Based Thread Safety

SovereignState uses lock for all mutable collections. Producer-consumer log queue via ConcurrentQueue.

🔄 Vector2 Everywhere

All positions use System.Numerics.Vector2 throughout — Raylib-CsLo GetMousePosition() returns this type.

🔌 Zero New NuGet Dependencies

All features built with existing deps: Raylib-CsLo 4.2.0.9, System.Text.Json, System.Runtime.Loader (in-box).
