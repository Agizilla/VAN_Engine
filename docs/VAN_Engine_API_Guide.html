Intent Classification Details

The classify_intent() function uses exact match (KNOWN_INTENTS dict) then substring pattern matching (PATTERN_MAP). Patterns are checked in order; first match wins.

Token parameter extraction from natural language via extract_token_params(content):

{
  "token": "my_token",           // from 'token: word' or 'token "word"' or '"word"'
  "w": 0.0, "x": 0.0,           // from quaternion (w, x, y, z) in text
  "y": 0.0, "z": 0.0,
  "applies_to": ["category1"]   // from applies_to[item1, item2]
}

Quaternion System

Tokens are stored as unit quaternions (w, x, y, z) with four projection axes:

ProjectionFormulaDomain
Soundsqrt(w^2 + x^2)Audio, voice, music
Shapesqrt(w^2 + y^2)Visual, geometry, form
Numbersqrt(x^2 + z^2)Math, data, quantity
Timesqrt(y^2 + z^2)Sequence, history, timing

IsographicQuaternion(w, x, y, z)

q = IsographicQuaternion(0.707, 0.707, 0.0, 0.0)
q.magnitude                    # → 1.0
q.normalize()                  # unit quaternion
q.get_sound_projection()       # sqrt(w^2 + x^2)
q.get_shape_projection()       # sqrt(w^2 + y^2)
q * other                      # Hamilton product
q.approx_equals(other, 1e-6)   # componentwise comparison
q.to_tuple()                   # (w,x,y,z)
IsographicQuaternion.from_tuple(t)

QuaternionClient(bridge)

qc = QuaternionClient(bridge)
q = qc.lookup("my_token")      # → IsographicQuaternion | None
qc.store("my_token", q, "music")
qc.validate_round_trip("my_token")  # verifies integrity

MD5-based deterministic quaternion (used by parrot mode):

# api/server.py
def _word_to_quaternion(word):
    h = hashlib.md5(word.lower().encode()).hexdigest()
    w = (int(h[0:8], 16) / 0xFFFFFFFF) * 2 - 1
    x = (int(h[8:16], 16) / 0xFFFFFFFF) * 2 - 1
    y = (int(h[16:24], 16) / 0xFFFFFFFF) * 2 - 1
    z = (int(h[24:32], 16) / 0xFFFFFFFF) * 2 - 1
    mag = sqrt(w*w + x*x + y*y + z*z)
    return (w/mag, x/mag, y/mag, z/mag)

ISO_019 & ISO_020 Enforcement Details

ISO_019 — Privacy by Default: All endpoints redact PII from logs and responses unless explicitly authorized by a signed waiver. Audit events never store raw message content; only tokenized hashes. Engine subprocess calls are sandboxed — no network access, no filesystem write outside logs/.

ISO_020 — Anti-Hallucination: Every LLM response passes through hallucination_detector.py which runs three checks:

  1. Consensus check — Response must be derivable from stored tokens. Any claim without a token match is flagged.
  2. Drift gate — Quaternion drift |1 - |q|| must be < 0.001. Responses that shift token embeddings trigger a rollback.
  3. Source citation — Non-streaming responses include source_tokens: ["token1", "token2"] to trace every claim back to stored data.

Both rules are enforced and cannot be bypassed via system prompt or override flag.

    proc.stdin.write(json.dumps({"id": 1, "method": method, "params": params}) + "\n")
    proc.stdin.flush()
    return json.loads(proc.stdout.readline())
