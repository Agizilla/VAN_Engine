🧠 Clawdia SAAS API

VAN_Engine ∞ Clawdia Collective — RESTful API for AI-powered transformations

ClawdiaBridge: http://localhost:8080 Brain API: http://localhost:11434 Mesh: P2P Gossip Protocol

🌐 Collective

POST
/clawdia/collective
Contribute anonymized session data to the AI Collective. Returns a personalized recommendation and available service list.
Body: sessions total_sessions total_messages donation_amount source
curl -X POST http://localhost:8080/clawdia/collective \
-H "Content-Type: application/json" \
-d '{"sessions":[],"total_sessions":3,"total_messages":42,"donation_amount":5,"source":"cli"}'
GET
/clawdia/inbox?target=<name>
Pull messages from a named queue. Auto-acks oldest message.
GET
/clawdia/db?source=<name>&limit=<n>
Query stored messages by source.

🖼️ Image Transform

POST
/api/v1/image/transform
Apply an audio-motif image processing algorithm. Accepts base64-encoded image. Returns base64 result + metadata.
Body: model (required) · image (base64, required) · params (object, optional)
Models: gray blur sharpen threshold negate resize rotate normalize median clahe hps kalman spectral mfcc vocoder echo psychoacoustic advanced-ocr
curl -X POST http://localhost:8080/api/v1/image/transform \
-H "Content-Type: application/json" \
-d '{"model":"spectral","image":"<base64>","params":{"windowSize":128,"noiseFloor":0.05}}'
POST
/api/v1/image/transform/upload
Upload a file directly instead of base64. Uses multipart/form-data.
Fields: model file params (JSON string)
curl -X POST http://localhost:8080/api/v1/image/transform/upload \
-F "model=gray" -F "file=@photo.jpg" -F 'params={"threshold":128}'
GET
/api/v1/image/file/{filename}
Retrieve a previously processed image by filename. Returns the image file directly.

🕸️ P2P Mesh (Phase 2.5)

GET
/api/v1/peers
List all registered P2P mesh nodes with trust scores and health status.
curl http://localhost:8080/api/v1/peers
GET
/api/v1/peer/status
Get mesh health: total peers, alive count, and whether mesh is ready for fault tolerance.
curl http://localhost:8080/api/v1/peer/status
POST
/api/v1/peer/register
Register a URL as a P2P mesh node. Once registered, collective contributions gossip to this peer.
Body: url (required) · label · trust_score
curl -X POST http://localhost:8080/api/v1/peer/register \
-H "Content-Type: application/json" \
-d '{"url":"http://192.168.1.42:8080","label":"my-server","trust_score":70}'
POST
/api/v1/peer/sync
Receive a P2P event broadcast from another mesh node. Events are forwarded to 2 random peers (gossip fanout).
Body: event (required) · origin · gossip_id · forwarded
DELETE
/api/v1/peer/{url}
Remove a peer from the mesh.

🎵 Audio Synthesis

POST
/audio/synthesize
Synthesize speech from text using the StyleTTS2 voice server. Supports emotion and voice intensity parameters.
Body: text (required) · source · emotion · intensity · voice_id
curl -X POST http://localhost:8080/audio/synthesize \
-H "Content-Type: application/json" \
-d '{"text":"Hello from the Collective","emotion":"neutral","intensity":1.0}'
GET
/audio/inbox
List all synthesized audio files in the queue.
GET
/audio/file/{id}
Download a synthesized audio file by its database ID.

🧠 Brain API (FastAPI — port 11434)

curl http://localhost:11434/v1/chat/completions \

🚀 Quickstart Examples

1. List all skills
curl http://localhost:8080/api/v1/skills | jq '.skills[].title'
2. Grayscale an image
curl -X POST http://localhost:8080/api/v1/image/transform -H "Content-Type: application/json" -d '{"model":"gray","image":"'$(base64 -w0 photo.jpg)'"}' | jq -r '.image' | base64 -d > output.png
3. Join the P2P mesh
curl -X POST http://localhost:8080/api/v1/peer/register -H "Content-Type: application/json" -d '{"url":"http://my-server:8080","label":"my-node"}'
4. Check mesh health
curl http://localhost:8080/api/v1/peer/status
5. Synthesize speech
curl -X POST http://localhost:8080/audio/synthesize -H "Content-Type: application/json" -d '{"text":"Hello Collective"}' --output speech.wav
