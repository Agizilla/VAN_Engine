PRD Format (v2.0)

YAML frontmatter with 8 required fields: task, slug, effort, phase, progress, mode, started, updated. Body has 4 sections: Context, Criteria (ISC checkboxes), Decisions (timestamped log), Verification (per-criterion evidence).

ConversationIDE Integration:

  • PRD creation: Each conversation/session should produce a lightweight PRD in MEMORY/WORK/{slug}/PRD.md. The /v1/session/export endpoint currently writes markdown — it should write PRD-format markdown.

Cloud Execution (Arbol)

Actions and pipelines run in two environments with identical behavior — local (bun runner) and cloud (Cloudflare Workers). Each action is a separate Worker. V8 Isolate Workers (lightweight, LLM actions) + Sandbox Workers (Docker containers, shell actions).

Settings.json Architecture

Single source of truth for all configuration: identity (daidentity, principal), environment, permissions, hooks, notifications, status line, spinner verbs, loadAtStartup files, dynamicContext toggles.

ConversationIDE Integration:
  • Principle 2 — Self-upgrading: The brain should be able to modify its own code via approved IPC handlers. The files:write handler already allows this — add guardrails.
  • Principle 6 — Code before prompts: Favor TypeScript handlers (written in src/main/) over brain prompts for deterministic logic. The brain handles flexible reasoning, but file operations, data transformation, and validation should be TS.
  • Principle 10 — CLI: Every IPC handler should have a corresponding CLI entry point (e.g., node dist/main/cli.js voice:notify '{"message":"hi"}').
  • Principle 15 — Science: The brain chat flow should support hypothesis testing — user states hypothesis, brain designs experiment, runs it, reports results.
  • Principle 16 — Permission to fail: The brain should return HALT_AND_CLARIFY freely when uncertain. The UI should render this distinctly from a normal response.
